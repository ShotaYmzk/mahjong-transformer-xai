実験本番では以下が必要になる:

```
# checkpoint 読み込み例
from models.mahjong_transformer_v2 import MahjongTransformerV2, MahjongTransformerConfig
```

game_state の作り方は `data/observation_schema.py` の `DatasetRow` を Tensor に変換すればよい（
`scripts/export_decision_report.py` の `build_batch()` 関数が参考になる）。

## `experiments/` 内の既存コードとの違い

混同しやすいため整理する。

| ファイル | 何をするか |
|---|---|
| `experiments/interventions/attention_masks.py` | モデル内部の attention **logits** に `-1e9` を注入し、モデル**出力（logits）**の変化（KL divergence, flip rateなど）を測る |
| `experiments/metrics/faithfulness.py` | `decision_flip_rate`, `kl_divergence`, `probability_drop`, `aopc` などの忠実性指標を計算する低レベルな関数群 |
| `experiments/visualize/attention_heatmap.py` | attention 行列を PNG または CSV に単純保存するユーティリティ |
| **`attention_patching.py`**（新規） | attention weights を **特徴グループ単位で集約・マスク**し、LLM 説明文の変化を BERTScore F1 とキーワードシフトで定量評価する |
| **`visualize_attention.py`**（新規） | マスク前後の特徴グループスコアを棒グラフで比較し、複数局面をヒートマップで表示する |

**既存の `experiments/` コードは「モデル出力の変化」を測る。**
**新しい `attention_patching.py` は「LLM説明文の変化」を測る。** 目的の層が違う。

## 仕様書と実際のモデルの差異

依頼仕様書に「`d_model=256, nhead=4`, `nn.TransformerEncoder`」と書かれていたが、
実際のコードは以下の通り（コードを読んで確認済み）:

| 仕様書の記述 | 実際のモデル（`models/mahjong_transformer_v2.py`）|
|---|---|
| `model.transformer_encoder`（`nn.TransformerEncoder`）| `model.blocks`（`ModuleList[HookedTransformerBlock]`）|
| `nn.TransformerEncoderLayer` / `nn.MultiheadAttention` | カスタム `HookedSelfAttention`（常に attn_weights を返す）|
| `d_model=256, nhead=4` | `d_model=128, n_heads=8` |
| `need_weights=True` の強制が必要 | 不要（`HookedSelfAttention` は常に weights を返す）|

新規ファイルはいずれも **実際のコード構造** に基づいて実装した。
`model.config` から設定を動的に読むため、checkpointの設定が変わっても動作する。

## 実際のXMLを使った実験（同日追記）

`experiments/run_attn_group_mask_experiment.py`（k=3 固定）と
`experiments/run_attn_k_sweep_experiment.py`（k=1〜5 スイープ）を作成し、
実際の天鳳 XML で実験を実施した。

- 使用 XML: `/home/ubuntu/Documents/tenhou_xml_2023/` から 5 ファイル、50 サンプル
- 全 k で KL(top) > KL(random) > KL(bottom) が成立（Faithfulness 仮説を支持）
- k=2 で Faithfulness Gap 最大（0.100）、**t 統計量の最大は k=5（t=2.74）**、**本文の主報告は k=3**（t=1.90、p=0.032、解釈・飽和のトレードオフ）
- **k=3 採用の根拠**: 有意（p<0.05）かつ上位 3 グループを同時にマスクできる最小 k、介入の飽和を避ける、XAI 慣例の「top-3」に合わせる（**「t 最大」ではない**）
- 詳細な結果・解釈・論拠は [experiment-attn-group-mask-2026-05-12.md](experiment-attn-group-mask-2026-05-12.md) を参照

## 研究進捗資料用ドラフト（数値・図を校正済み）

スライド用に貼る場合の要約。**条件C** はスクリプトにより数値が二通りあるので表を分けた。

### Attention Patching（指示1）

- `attention_patching.py` — 忠実性評価クラス本体
- `visualize_attention.py` — 可視化ヘルパー

**用語**: Attention（参照の強さ）／Attention Patching（重みを意図的に書き換えて振る舞いを測る）／Faithfulness（説明と実際の参照の一致度）。

**実験**

| 実験 | 内容 | データ |
|------|------|--------|
| 実験1（`run_attn_group_mask_experiment`） | k=3 で上位・下位・ランダムマスク。**random は各局面 1 回** | 50 局面 |
| 実験2（`run_attn_k_sweep_experiment`） | k=1〜5 で同型の比較＋t 検定。**random は各局面 5 回平均** | 50 局面 × k（1〜5）× 3 条件 |

**k=3・実験2（図・`attn_k_sweep` JSON と一致）**

| 条件 | 平均 KL | Flip Rate |
|------|---------|-----------|
| A 重要グループ | **0.1396** | **14.0%** |
| C ランダム（5×平均） | **0.0993** | **10.8%** |
| B 重要でないグループ | **0.0000** | **0.0%** |

**実験1 JSON（`attn_group_mask`）の条件Cのみ**: KL **0.0895**, Flip **8.0%**（random 1 回のため）。**スライドで AOPC 等の図と並べるときは上段の実験2定義を使う。**

**Faithfulness Gap 図が金で k=2 になる理由**: 図は「平均 KL(top−random)」の**最大の k** を強調しているため（今回 k=2）。採用 k=3 は **解釈性・上位3グループ同時マスク・KL飽和の手前**などの別基準（**t 統計量の最大は k=5**）。更新した図では **緑枠 = Adopted k=3**。

**k スイープ（片側 t、top vs random）**: k=1 は p=0.064。**t が最大なのは k=5**。資料では **k=3**（t=1.90, p=0.032）を主に採用と記載（過剰介入を避けるトレードオフ）。

**グループ重要度（50 サンプル平均）**: SafetyVsOthers 70.6%、TileEfficiency 17.6%、PointSituation 7.2%、他は当該サンプルでほぼ 0。
