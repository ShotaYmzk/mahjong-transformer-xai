## 実際に使ったデータ・XML・チェックポイント

**今回はランダムテンソルで動作確認のみ。実際のXMLやcheckpointは使っていない。**

実験本番では以下が必要になる:

```
# checkpoint 読み込み例
from models.mahjong_transformer_v2 import MahjongTransformerV2, MahjongTransformerConfig
import torch

ckpt = torch.load("outputs/impl1/hdf5_10epoch.pt", map_location="cpu")
cfg = MahjongTransformerConfig(**ckpt["config"])
model = MahjongTransformerV2(cfg)
model.load_state_dict(ckpt["model_state"])
model.eval()
```

game_state の作り方は `data/observation_schema.py` の `DatasetRow` を Tensor に変換すればよい（
`scripts/export_decision_report.py` の `build_batch()` 関数が参考になる）。

## `experiments/` 内の既存コードとの違い

混同しやすいため整理する。

| ファイル | 何をするか |
|---|---|
| `experiments/interventions/attention_masks.py` | モデル内部の attention **logits** に `-1e9` を注入し、モデル**出力（logits）**の変化（KL divergence, flip rateなど）を測る |
| `experiments/metrics/faithfulness.py` | `decision_flip_rate`, `kl_divergence`, `probability_drop`, `aopc` などの忠実性指標を計算する低レベルな関数群 |
| `experiments/visualize/attention_heatmap.py` | attention 行列を PNG または CSV に単純保存するユーティリティ |
| **`attention_patching.py`**（新規） | attention weights を **特徴グループ単位で集約・マスク**し、LLM 説明文の変化を BERTScore F1 とキーワードシフトで定量評価する |
| **`visualize_attention.py`**（新規） | マスク前後の特徴グループスコアを棒グラフで比較し、複数局面をヒートマップで表示する |

**既存の `experiments/` コードは「モデル出力の変化」を測る。**
**新しい `attention_patching.py` は「LLM説明文の変化」を測る。** 目的の層が違う。

## 仕様書と実際のモデルの差異

依頼仕様書に「`d_model=256, nhead=4`, `nn.TransformerEncoder`」と書かれていたが、
実際のコードは以下の通り（コードを読んで確認済み）:

| 仕様書の記述 | 実際のモデル（`models/mahjong_transformer_v2.py`）|
|---|---|
| `model.transformer_encoder`（`nn.TransformerEncoder`）| `model.blocks`（`ModuleList[HookedTransformerBlock]`）|
| `nn.TransformerEncoderLayer` / `nn.MultiheadAttention` | カスタム `HookedSelfAttention`（常に attn_weights を返す）|
| `d_model=256, nhead=4` | `d_model=128, n_heads=8` |
| `need_weights=True` の強制が必要 | 不要（`HookedSelfAttention` は常に weights を返す）|

新規ファイルはいずれも **実際のコード構造** に基づいて実装した。
`model.config` から設定を動的に読むため、checkpointの設定が変わっても動作する。

## 実際のXMLを使った実験（同日追記・2026-05-12 に再計測）

`experiments/run_attn_group_mask_experiment.py`（k=3 固定）と

スイープ結果の読みどころ：**図5（`figure/attn_k_sweep_aopc_curve.png`）**、**図6（`figure/attn_k_sweep_faithfulness_gap.png`）**、**図9（`figure/attn_k_sweep_pvalues.png`）**。k=3 固定では **図1（`figure/attn_mask_kl_comparison.png`）** と **図2（`figure/attn_mask_fliprate.png`）**。

定量の要約（詳細・表は実験記録を参照）:

- Faithfulness：`KL(top) > KL(random) > KL(bottom)` の秩序が全 k で概ね維持（図5）。
- Gap 平均最大は **k=2**（図6の金色）；SNR 最大は **k=1**（図7）；**対応あり t は k=5 で最大**（図9）。
- **主報告 k=3** は、p だけでの選択ではなく **上位 3 解釈グループを同時マスクできる最小単位**・介入の過大化回避・慣例のため（実験記録の論拠セクション）。

## 研究進捗資料用ドラフト（数値・図）

**条件C** は実装により二通りある（必ずどちらの定義かをスライドに書く）。

| 実験 | 内容 | データ |
|------|------|--------|
| 実験1（`run_attn_group_mask_experiment`） | k=3 で上位・下位・ランダムマスク。**random は各局面 1 回** | 150 局面 |
| 実験2（`run_attn_k_sweep_experiment`） | k=1〜5。**random は各局面 5 回平均** | 150 × k × 条件 |

**k=3・実験2（スイープ JSON と図5〜図9で整合）**

| 条件 | 平均 KL | Flip Rate |
|------|---------|-----------|
| A 重要グループ | **0.0958** | **約 14.7%** |
| C ランダム（5×平均） | **0.0438** | **約 8.5%** |
| B 重要でないグループ | **約 0.0000** | **0.0%** |

**実験1 JSON（random 1 回のみ）の条件 C**: KL **0.0597**, Flip **8.0%**。**AOPC や図5と並べるときは実験2の行を使う。**

**図6（`figure/attn_k_sweep_faithfulness_gap.png`）と図7（`figure/attn_k_sweep_snr.png`）**: それぞれ **Gap 最大**と **SNR 最大**の k が金色になり、両者は一致しない（本データでは Gap→k=2、SNR→k=1）。**緑枠の Adopted k=3** は設計選択。

**k スイープ（片側 t）**: **t 最大は k=5**（図9）。n=150 ではすべての k で p が非常に小さいため、報告粒度は Gap・解釈とセットで議論する。

**グループ重要度（実験1 baseline、150 平均）**: SafetyVsOthers 約 0.675、TileEfficiency 約 0.176、PointSituation 約 0.114（残りカテゴリは小さめ）。分布の全体像は **図4（`figure/attn_group_heatmap.png`）**。
